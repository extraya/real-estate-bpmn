# Үл хөдлөх хөрөнгийн зуучлалын процесс (BPMN)

**Tool:** ADONIS BPM Suite
**Notation:** BPMN 2.0

## Тайлбар

Энэхүү диаграм нь үл хөдлөх хөрөнгө зуучлалын агентлагийн бизнес процессыг зуучлалын гэрээ баталгаажснаас эхлээд худалдан авагчтай хаалт (closing) хийх хүртэлх бүхэл дамжлагыг харуулсан BPMN загвар юм.

Процесст оролцогч 4 тал (pool) байна:
- **ҮХХ эзэн** (Property owner) — зуучлалын гэрээ баталгаажуулж, эцсийн шатанд гэрээнд гарын үсэг зурна
- **Оффис** — дотроо **Агент** болон **Админ** гэсэн 2 lane-тай, гол ажиллагааг гүйцэтгэнэ
- **Худалдан авагч тал** (Buyer) — сонирхол илэрхийлж, санал тавьж, хаалт хийнэ

## Процессын үндсэн үе шатууд

1. **Материал хүлээн авах** — зуучлалын гэрээ баталгаажсаны дараа агент шаардлагатай материал, зураг цуглуулна
2. **Бэлтгэл (лист бүртгэх/батлуулах)** — цэвэрлэгээ шаардлагатай эсэхийг шалгаж, лист бэлтгэн батлуулна
3. **Маркетинг** — parallel gateway ашиглан олон сувгаар (Facebook, портал сайтууд, брошур, open house, тухайн бүсийн агентууд гэх мэт) зэрэг зарлана
4. **Сонирхогчидтой ажиллах** — холбогдож ирсэн сонирхогчидтой танилцах хүсэлт, худалдан авах саналыг хүлээн авч, шаардлагатай бол дахин маркетинг руу буцна
5. **Гэрээ хийх** — тал бүр гарын үсэг зурж, гэрээг баталгаажуулна
6. **Хаалт** — санхүүгийн үйл явц, нэхэмжлэл илгээх, эцсийн хаалт хийх

## Онцлох BPMN элементүүд

- **Parallel gateway (+)** — маркетингийн олон суваг зэрэг эхлэх, мөн зэрэг дуусахыг загварчилсан
- **Exclusive gateway (◇)** — "Цэвэрхэн эсэх?", "Худалдан авах санал?", "Нэмэлт үйл ажиллагаа шаардлагатай?" гэх мэт шийдвэр цэгүүд, Тийм/Үгүй салаалалттай
- **Timer event** — "Үйл ажиллагааны мэдээллийг илгээх" дараах хугацааны хүлээлт
- **Message end event** — "Хэрэглэгчээс чанарын судалгаа бөглүүлэх" бол процессын төгсгөлд харилцагчтай холбогдох message event
- **Data objects** — Гэрээ болон хавсралт, Тайлан, Худалдан авах санал зэрэг баримт бичгүүдийг тодорхой зурсан
